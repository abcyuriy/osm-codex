# TableViewHeaderFooterView

<MediaScroller items={items} />

## Назначение
`TableViewHeaderFooterView` — компонент для оформления **заголовков и подвалов секций списка**. Используется для структурирования контента, добавления контекстных пояснений и размещения вторичных действий на уровне секции, а не отдельной строки.

Компонент применяется внутри списков и работает как логическая «обёртка» для группы элементов.

---

## Способы использования

### Заголовок секции (Header)
Используется для:
- обозначения начала логического блока;
- указания темы или категории;
- размещения действия, относящегося ко всей секции.

### Подвал секции (Footer)
Используется для:
- поясняющего текста;
- инструкций;
- справочной или юридической информации;
- перехода к дополнительным материалам.

---

## Параметры

### Контент
- `title` — основной заголовок секции.
- `subtitle` — дополнительный текст под заголовком (опционально).
- `footerText` — текст подвала секции.
- `secondaryText` — расширенное описание (многострочное).

### Действия
- `accessoryAction` — интерактивный элемент справа (кнопка/ссылка).
- `accessoryStyle` — визуальный стиль действия (secondary / primary).
- `accessoryLabel` — текст действия (например, `See more`, `Learn more`).

### Раскрытие
- `isExpandable` — секция может раскрываться и сворачиваться.
- `isExpanded` — текущее состояние секции.
- `disclosureIndicator` — индикатор раскрытия (`>` или `˅`).

---

## Варианты отображения

### Header-варианты

1. `Default Header`  
   Заголовок без действий и индикаторов.

2. `Header with Accessory`  
   Заголовок с вторичным действием справа.

3. `Header with Primary Accessory`  
   Заголовок с акцентным действием.

4. `Expandable Header`  
   Заголовок с индикатором раскрытия секции.

5. `Primary Header`  
   Увеличенный заголовок для верхнего уровня иерархии.

---

### Footer-варианты

1. `Default Footer`  
   Краткий поясняющий текст.

2. `Multi-line Footer`  
   Расширенное описание в несколько строк.

3. `Footer with Link`  
   Текст с интерактивной ссылкой (`Learn more`).

4. `Truncated Footer`  
   Ограниченный по количеству строк текст с усечением.

5. `Footer with Custom Accessory`  
   Подвал с дополнительным элементом управления.

---

## Состояния (State Machine)

### Базовые состояния
- `Default` — обычное отображение.
- `Highlighted` — активное состояние при взаимодействии.
- `Disabled` — неинтерактивное состояние.

### Состояния раскрытия
- `Collapsed` — содержимое секции скрыто.
- `Expanded` — содержимое секции отображается.

### Модификаторы
- `WithAccessory` — отображается действие.
- `WithDisclosure` — отображается индикатор раскрытия.
- `WithLink` — текст содержит интерактивную ссылку.

---

## Поведение

- Высота компонента адаптируется под объём текста.
- Заголовок и подвал не перекрывают контент секции.
- Действия справа не влияют на выравнивание текста.
- При раскрытии/сворачивании обновляется только контент секции.
- Секция сохраняет состояние при прокрутке списка.

---

## Ограничения и правила

- Заголовок не должен дублировать текст строк внутри секции.
- В одной секции рекомендуется не более одного действия.
- Подвал не используется для критичных действий.
- Раскрывающиеся секции применяются только при реальной необходимости экономии пространства.
- Компонент не используется вне контекста списка.

---

## Рекомендации по использованию

Подходит для:
- группировки настроек;
- каталогов и списков параметров;
- справочных и обучающих экранов;
- длинных списков с пояснениями.

Не рекомендуется для:
- одиночных элементов;
- экранов без секционной структуры;
- сценариев, где действие относится к конкретной строке.
